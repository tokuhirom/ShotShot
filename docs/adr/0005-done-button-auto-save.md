# ADR 0005: Done ボタンによる自動保存 UX

## Status

Accepted

## Context

スクリーンショット編集後の保存フローについて、ユーザー体験を検討した。

従来のアプローチ:
- 「保存」ボタンと「クリップボードにコピー」ボタンを別々に配置
- ユーザーが明示的に保存先を選択

Skitch 風のアプローチ:
- 編集完了時に自動的に保存
- 1 アクションで完結

## Decision

「Done」ボタンを実装し、押下時に以下の処理を自動実行する:

1. ファイルに保存（設定された保存先に自動保存）
2. クリップボードにコピー
3. ウィンドウを閉じる

キーボードショートカット:
- `Enter` キーで Done
- `Escape` キーでキャンセル

## Consequences

### Positive
- 1 アクションで完結するシンプルな UX
- スクリーンショット撮影から共有までの時間を短縮
- Skitch や CleanShot X などの既存ツールと同様の操作感

### Negative
- 「保存せずにクリップボードだけにコピー」ができない
- 保存先を毎回選択できない（設定で事前に指定する必要がある）

### Future Considerations
- Option + Enter で「クリップボードのみ」などの拡張が可能
- 保存完了時の通知表示
